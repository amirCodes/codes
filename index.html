Testing in the cloud

Testing as a Service in the Cloud Environments...

Abstract—Cloud computing count as an important part of IT and software developments in industries. Cloud computing is used to deliver the hosted services throughout the internet. Testing tools have been used to test mobile applications, web applications and the cloud based software systems. In the existing paper, it is not providing the quality of services in the effective manner. In this paper, we are going to focused on integrating the hypotheses for getting the quality of services, in terms of performance and latency at different locations, times, and number of users. By using testing as a service in the cloud environments we will be able to minimize the time, cost spending on service and come up with an efficient service for testing. The results show that using cloud environment and cloud services for testing is more flexible and provides unlimited resources. In the same times, we find out that there are differences between the performance of services (google drives) based on different locations. While there is not that much differences based on the time of testing and number of users. Our observations show that if we do the testing with more number of users we will gain more realistic result and data which is the future work for this paper.

Cloud computing is the evolution of new technologies and over the years, the term cloud computing has become increasingly important in the IT industry. In this context, cloud is used as a service where we can share the resources, software, information and provide other services like testing, storing data over the Internet.
